?utm_source=Google&utm_medium=Shopping (veja detalhes sobre parâmetros de campanha personalizada nas URLs)

Caracteres como ?, = e & devem ser codificados.

Exemplo:

? = %3F
= = %3D
& = %26

Para adicionar nos links do XML os parâmetros ?par1=aaaaa&par2=bbbbb&par3=ccccc, deve-se passar para o XMLProdutos.asp o parâmetro AddParURL. Exemplo:

&AddParURL=%3Fpar1%3Daaaaa%26par2%3Dbbbbb%26par3%3Dccccc

Com padrão de URL amigável:

http://www.sualoja.com.br/XMLProdutos.asp?IDLoja=99999&AddParURL=%3Fpar1%3Daaaaa%26par2%3Dbbbbb%26par3%3Dccccc&Format=0

Sem padrão de URL amigável:

http://www.sualoja.com.br/XMLProdutos.asp?IDLoja=99999&AddParURL=%26par1%3Daaaaa%26par2%3Dbbbbb%26par3%3Dccccc&Format=0&ua=0

Para URL amigável, os parâmetros adicionais devem começar com ?. Caso a URL não seja amigável, devem começar com &.

Link para o produto no XML (com URL amigável):

http://www.sualoja.com.br/prod,idloja,99999,idproduto,11111,texto?par1=aaaaa&par2=bbbbb&par3=ccccc

Link para o produto no XML (sem URL amigável):

http://www.sualoja.com.br/listaprodutos.asp?idloja=99999&idproduto=11111&q=texto&par1=aaaaa&par2=bbbbb&par3=ccccc

É possível informar estes parâmetros adicionais no campo Parâmetros adicionais (URL) da página de geração de XML no site administrativo da loja. (menu Produtos > XML). Neste caso, os caracteres &, = e ? não precisam ser codificados pois isso ocorrerá automaticamente.
