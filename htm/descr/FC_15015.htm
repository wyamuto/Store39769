Custom URLs must be alphanumeric, with no accents. Spaces should be replaced with hyphens. URLs must be unique, since the store should not have two pages with the same address.

To enable this feature, go to the Navigation and SEO section of the STORE CONFIGURATION page of the control panel. Check the Custom URLs field and enter the terms that will be used at the beginning of the custom URL that lists the products in a category and news. These terms will be displayed between bars / in the page address.

  • Path to Categories: When filling in this field with brand, the custom URL for products in each category of the store will be /brand/category-name. Access the tab for each category to set your custom URL. When you visit the URL /brand you will be presented with a page with the list of store categories.
  • Path to News: When filling this field with news, the custom URL to list the store news will be /news. The individual news will be accessed by the /news/url-news, where url-noticia is what was filled in the URL field of the news story.

ATTENTION: Before customizing product and news URLs, take the following test:

1 Step

Go to a category listing and enter your custom URL.

2 Step

Go to the custom URL for the category you created in the previous step and access the links on this page. Make sure there are no ERROR 404 on these links and that everything is OK in the store features. Click here for guidelines in the Fixed Issues section.

ATTENTION: The tests should also be done in the mobile version and the Facebook version of the store.

Turn off the custom URL feature if a problem is identified. Please contact our support team if you have any questions.

3 Step

After performing the tests and making sure that no errors were found, disable the custom URLs feature and run the following utilities to create all store URLs:

  • Defines custom product URLs
  • Defines custom category URLs
  • Defines custom news URLs (only if the store has news stories)

Obs: URLs can also be created individually via the tab for each product, category and news.


4 Step

After running the utilities from the previous step, access the following reports to list the URLs that were created:

  • Products registered by custom URL and Title
  • Categories registered
  • News registered by custom URL and Title (only if you have registered news)

Obs: Make sure there are no duplicate URLs and all of them match the format you want. Corrections should be made in the tab of each product, category and news.

ATTENTION

Enabling this feature involves re-indexing search sites for store and product URLs. Accesses to old URLs will automatically be redirected to new custom URLs via redirect 301. Therefore, do not enable this feature without making sure that everything is fine.

After registering custom URLs, avoid changing them again so that 404 error (page not found) does not occur in hits made to the previous custom URL. When changing the page URL, consider that it is likely that Google has already indexed the previous URL, which will no longer exist.

Correction of problems encountered

Check the HTML and javascript files that were customized because relative links that do not start with slash (/) will no longer work.

Example:

  • Incorrect link: <a href="prod,offers,1,offers.htm">
  • Correct Link: <a href="/prod,offers,1,offers.htm">

Another example is the shopping cart function:

  • Incorrect link: oXMLHTTP.open("GET","XMLCart.asp",false);
  • Correct Link: oXMLHTTP.open("GET","/XMLCart.asp",false);

If the store has static links for categories, change them as follows.

  • Incorrect link: <a href="/prod,catid,99999,apple">Appel</a>
  • Correct Link: <a href="/category/apple">Apple</a>

Obs: Tests should also be done on Facebook, Mobile and alternative A / B test designs if the store uses any of these features.
